# VoC 데이터 인터페이스 요청서

## 배경 및 요청 사유

현재 `solution_if_voc` 테이블은 DATALAKE API를 통해 **VOC건수·불만건수·실사용자수** 3개 지표를 월별 집계(CNT) 형태로 수신 중입니다.

기존 데이터는 월별 합산값(상품명 × 지표유형 × 연월 → 건수 1개) 구조이므로, 개별 건 단위의 리드타임·지연 여부 분석이 불가합니다.

이에 사업건전성 대시보드의 신규 VoC 지표(개선건수·리드타임·지연건수) 산출 및 상품별·채널별·처리구분별 세부 분석을 위해 CS 대시보드(SCC) 원천의 **건별 RAW 데이터** 제공을 요청합니다.

---

## 1. 요청 개요

| 항목 | 내용 |
|------|------|
| 요청 목적 | Solution Biz Health Check 대시보드 VoC 지표 산출을 위한 원천 데이터 제공 |
| 활용 시스템 | 사업건전성 대시보드 (신규 테이블 적재) |
| 원천 시스템 | CS 대시보드 (SCC) |

---

## 2. 인터페이스 방식

기존 데이터 제공 창구(`/datalake/QualitySummary`)는 월별 집계 전용 구조로, 건별 RAW 데이터(12개 필드)와 데이터 단위·구조가 근본적으로 달라 기존 테이블에 단순 추가가 불가합니다.

따라서 기존 창구와 테이블은 그대로 유지하고, **신규 데이터 제공 창구(API) 추가 개발**을 요청합니다. 기존 운영에 영향 없이 신규 창구를 기존 배치(1일 1회)에 얹는 형태로 협의를 희망합니다.

| 항목 | 기존 | 신규 요청 |
|------|------|------|
| 창구 | `/datalake/QualitySummary` | 신규 RAW 조회용 창구 추가 개발 |
| 테이블 | `solution_if_voc` (현행 유지) | 신규 테이블 추가 |
| 데이터 단위 | 월별 집계 (CNT) | 건별 RAW |
| 조회 범위 | 1년 이내 | **제한 해제 요청** (2023년 1월~현재, 약 3년치) |
| 제공 주기 | 1일 1회 | 매월 1회 Full 재집계 |

---

## 3. 데이터 제공 조건

| 항목 | 내용 |
|------|------|
| 제공 형태 | RAW 데이터 (집계는 수신 측에서 직접 수행) |
| 제공 범위 | 2023년 1월 ~ 제공 시점 당월 전체 |
| 제공 주기 | 매월 1회 |
| 적재 방식 | Full 재집계 (기존 데이터 전체 삭제 후 재적재) |

> **Full 재집계 사유:** 지연 여부는 집계 시점 기준으로 판단되므로 과거 데이터도 매월 갱신 필요. 3년치 시계열 트렌드 분석 목적으로 전체 기간 데이터를 매월 제공 요청.

## 6. 산출 예정 지표 (참고)

| # | 지표명 | 계산식 | 비고 |
|---|--------|--------|------|
| 1 | 접수월별 개선 요청 건수 | COUNT(VOC ID) | 전체 건 |
| 2 | 접수월별 평균 처리 리드타임 | AVG(리드타임(시간)) | 진행단계='완료'인 건 |
| 3 | 접수월별 지연 건수 | COUNT((진행단계='완료' AND 완료일자 > 예정일자) OR (진행단계≠'완료' AND 현재 > 예정일자)) | 완료 지연 + 미완료 지연 모두 포함 |

> **지연 정의 관련:** 원천의 지연여부 플래그(Y/N)는 미사용. ① 완료됐으나 완료일자가 예정일자를 초과한 건 + ② 미완료이면서 현재 시점이 예정일자를 초과한 건을 모두 지연으로 자체 정의.

## 4. 기존 인터페이스 수정 — 신규 GUBUN 추가

| GUBUN | 설명 | 산출 조건 |
|-------|------|----------|
| **개선요청건수** | 접수 기준 개선 요청 건수 | 요구유형 = '개선', 완료 여부 무관 |
