AI 구축·운영 검증 프로젝트

프로젝트 구조서 — 전제·목표·환경·팀·제약·사전작업 전체 정리
핵심 질문 웹디자이너·Tableau 개발자·DB 개발자 없이,
AI만으로 사업건전성 대시보드를 구축·운영·유지보수할 수 있는가?

단계검증 프로젝트
검증 가설

AI를 활용하면 전문 개발 인력(Tableau 개발자·DB 전문가·웹디자이너) 없이도
기획자가 사업건전성 대시보드를 직접 구축·운영·유지보수할 수 있다.

이 가설이 검증되면 → 실제 Web/WAS 서버 기반 프로젝트로 전환. 검증 실패 또는 부분 성공 시 → 결과에 따라 범위 조정 후 재설계.
1

현황 — 지금 어떻게 운영하고 있는가

현재 기술 스택
  • 시각화: Tableau
  • 데이터베이스: Impala DB
  • 대상: 사업건전성 대시보드
현재 운영의 문제
  • Tableau 개발자 상시 필요
  • DB 전문가 상시 필요
  • 웹디자이너 상시 필요
  • → 요구사항 적기 반영 어려움
2

목표 — 무엇을 바꾸려 하는가

AS-IS (현재)
Tableau (시각화)
Impala DB (데이터)
전문가 팀 (개발·유지보수)
느리다 · 비용이 크다 · 기획자 통제권 없음
→ 대체
TO-BE (목표)
AI (설계·개발·유지보수 주도)
Web/WAS Server (시각화)
Impala DB (데이터, 유지)
빠르다 · 기획자가 직접 주도 · 전문가 의존 낮음

Tableau를 Web/WAS Server + 필요 애플리케이션으로 대체. Impala DB는 그대로 유지. 핵심 변화는 "누가 개발하는가" — 전문가에서 AI(+기획자)로.

3

검증 방식 — 어떻게 검증하는가

검증 프로젝트 (현재)

Windows PC + Impala DB

  • 실제 Web/WAS 서버 대신 사내 Windows PC 활용
  • Impala DB 접근 가능한 PC를 개발·검증 환경으로 사용
  • 소규모·저비용으로 가설 먼저 검증
실제 프로젝트 (검증 후 전환)

Web/WAS Server + Impala DB

  • 검증 성공 시 실제 서버 환경으로 전환
  • 운영 수준의 인프라 및 보안 체계 적용
  • 조직 내 공식 프로젝트로 승인·추진

검증의 핵심 질문 3가지
  • 구축: AI가 요구사항을 받아 웹 대시보드를 실제로 만들 수 있는가?
  • 운영: AI가 데이터 변경·오류 대응을 기획자와 함께 처리할 수 있는가?
  • 유지보수: AI가 요구사항 변경 시 수정·배포를 신속하게 반영할 수 있는가?

7

검토 필요 사항 — 아직 결정되지 않은 것

①

Claude Enterprise 3인 협업 환경 설정

3인이 동일 프로젝트 컨텍스트를 공유하는 방법, Claude Enterprise의 팀/프로젝트 기능 활용 방식 확인 필요

②

AIPro/FabriX 기능 범위 확인

사내 AI 도구(AIPro/FabriX)가 코드 생성·SQL 생성·파일 업로드·스케줄 실행 중 무엇을 지원하는지 미확인. 지원 범위에 따라 과제 추진 방식이 달라짐

③

Windows PC에서 웹 아티팩트 배포 방식

AI가 생성한 HTML/JS 코드를 어디에 배포하여 실제로 확인할 것인가. 실제 Web 서버가 없는 상태에서의 대안 환경 확정 필요

④

3인 역할 분담 확정

현재 동료 2인의 역할이 미정. 데이터/프론트/PM으로 분리할 것인지, 또는 다른 분담 방식을 택할지 킥오프 전 합의 필요

⑤

사전 작업 결과물의 AI 입력 방식

GitHub에 저장된 계획서·DB 재설계·와이어프레임을 AI에게 어떻게 전달할 것인가. 파일 업로드, URL 참조, 텍스트 복사 중 어떤 방식이 현실적인지 확인 필요
