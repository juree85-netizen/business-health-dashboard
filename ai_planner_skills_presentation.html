스킬은 "보고서를 빨리 쓰는 도구"가 아니다. 기획자가 대상을 정의하고, 구조를 파악하고, 문제를 규정하는 — 보고서 이전의 사고 단계를 설계한다.
Framework
기획의 사고 단계 — 4단계
01
정의
DEFINE
다루는 대상이 무엇인지 명확히 한다. 개념의 범위, 시장의 경계, 경쟁 구도를 설정한다.
concept-map
02
조사
INVESTIGATE
대상의 구조와 구성 요소를 파악한다. 어떤 플레이어가 있고, 어떤 관계로 연결되어 있는지 살핀다.
concept-map
03
조망
SURVEY
전체 지형에서 우리가 어디에 있는지, 문제가 비즈니스 모델의 어느 블록에 위치하는지 바라본다.
business-diagnosis
04
규정
DEFINE THE PROBLEM
문제의 유형과 우선순위를 확정한다. 해결 방향과 권고안을 기획자의 언어로 규정한다.
business-diagnosis
Skills
두 가지 스킬
/concept-map
개념 설명 지형도
개념·기업·시장·기술을 구조적으로 조망하는 지형도. 내가 다루는 대상의 전체 생태계를 한눈에 파악한다.
1
대상 개념/기업/시장의 정의와 범위 설정
2
핵심 구성요소·경쟁자·연관 개념 분류
3
시장 내 위치와 관계 구조 시각화

처음 접하는 기업·시장을

/concept-map
기업의 사업 구조, 경쟁 포지션, 시장 내 위치를 지형도로 파악 — 내가 어디를 보고 있는지 먼저 확인
/business-diagnosis
기업이 가진 사업 문제를 BMC로 위치 특정 → 어떤 블록이 취약한지, 어떤 방향으로 봐야 하는지 규정
보고서 작성 전, 기획자만의 관점과 가설이 먼저 생긴다
CASE 02 — 업무 배경지식 습득
생소한 개념·기술·산업을
업무에 활용 가능한 수준으로 이해할 때
/concept-map
개념의 정의·범위·하위 요소·관련 기술을 지형도로 정리 — 용어와 구조를 내 것으로 만드는 단계
/business-diagnosis
해당 개념이 우리 사업의 어느 문제와 연결되는지 진단 — 배경지식을 실무 관점으로 전환
"알고 있다"에서 "어떻게 쓸 것인가"로 사고가 확장된다
Implication
시사점 — 기획자의 역할이 달라진다
AI 스킬은 더 빨리 쓰기 위한 도구가 아니다.
기획자가 대상을 정의하고, 구조를 파악하고, 문제를 규정하는
— 스스로 사고하는 단계를 AI와 함께 설계하는 방식.
🗺️
산출물 이전에
관점이 먼저 생긴다
보고서를 쓰기 전, 기획자 자신의 시각과 가설이 먼저 형성된다. AI가 아닌 내가 방향을 잡는다.
🧩

🤝

파트너가 된다
