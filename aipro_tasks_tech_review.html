문서 목적

사내 보안 정책으로 인해 Claude Code 대신 AIPro를 활용하여 사업건전성 대시보드 고도화를 추진한다. 본 문서는 추진 예정인 4개 과제 각각에 대해 배경, 목표, 프로세스 변화, 그리고 AIPro로 구현하기 위해 사전에 해결해야 할 기술적 문제를 정리한 것으로, 개발자 미팅 및 내부 검토를 위한 자료로 활용한다.

4개 과제는 독립적으로 추진되나 일부 공통 기술 문제를 공유하므로, 공통 사항은 문서 후반부에 별도 정리하였다.

TASK 02

AI 기반 데이터 모델 개선

DB 전문가 2인·2~3개월 투입 대신 AI가 ERD·테이블 정의·SQL 쿼리를 설계·생성

TASK 03

AI 시안 생성 + 데이터 시뮬레이션

설계 단계에서 AI가 시안을 생성하고 가상 데이터로 KPI 로직을 사전 검증

TASK 04

자동 분석 + 인사이트 리포트 푸시

AI가 지표 변화를 감지하고 원인 분석 후 인사이트 리포트를 메일로 자동 발송

TASK 01

AI 기반 웹 대시보드 재구축

추진 배경 및 문제

현재 사업건전성 대시보드는 Tableau 기반으로 운영되고 있으며, 상품 및 지표 추가·지속적인 개선을 위해 Tableau 개발 전문가와 DB 전문가가 상시적으로 필요한 구조다. 이로 인해 기획자가 요구사항을 정의하더라도 개발 리소스 확보까지 대기 시간이 발생하고, 사소한 변경에도 전문 인력이 투입되어야 한다.

목표

AI를 활용하여 사업건전성 대시보드를 웹 기반으로 재구축하고 운영 가능성을 검증한다. 이를 통해 Tableau 의존도를 낮추고, 개발 리소스와 개발 기간을 단축하고자 한다.

프로세스 변화
기존 프로세스
  1. 기획자가 비즈니스 요구사항 도출
  2. Tableau 개발자·DB 전문가가 기술 요구사항 및 설계
  3. Tableau 개발자가 화면 개발 / DB 전문가가 SQL 쿼리 개발
변경 프로세스
  1. 기획자가 비즈니스 요구사항 도출
  2. AI가 기술 요구사항 정의 및 설계 (전문가 검증·수정)
  3. AI가 프론트/백엔드/SQL 쿼리 개발 (전문가 검증·수정)
해결해야 할 기술 문제
📋

컨텍스트 전달 방법

기존 Tableau 대시보드 구조, 지표 정의서, DB 스키마 등 대량의 사전 정보를 AI에게 효과적으로 전달하는 방법이 필요하다. 정보량이 많을 때 AI가 누락 없이 반영하는지 검증이 필요하다.

확인 필요: AIPro에서 대용량 문서 업로드 및 참조 기능 지원 여부
⚙️

코드 실행 및 검증 환경

AI가 생성한 프론트엔드·백엔드·SQL 코드를 실제로 실행하여 동작을 확인해야 한다. AIPro 내에서 코드 실행 환경이 제공되는지, 아니면 별도 개발 환경을 연동해야 하는지 확인이 필요하다.

확인 필요: AIPro 코드 실행 환경 지원 범위 (프론트/백엔드/SQL)
🔄

반복 수정 시 맥락 유지

1회 생성으로 완성되지 않으며 검토-수정-재생성 사이클이 반복된다. 수정 요청이 누적될 때 AI가 이전 맥락을 정확히 유지하면서 생성 결과물의 일관성을 보장하는지 확인이 필요하다.

확인 필요: AIPro의 긴 대화 맥락 유지 한계 및 세션 관리 방식
🔐

사내 DB 접속 권한

AI가 생성한 SQL을 검증하거나 실제 데이터로 테스트하려면 AIPro가 사내 DB에 접근할 수 있어야 한다. 보안 정책상 접근 가능 범위와 인증 방식 확인이 선행되어야 한다.

확인 필요: AIPro에서 사내 DB 연결 허용 여부 및 접근 방식
TASK 02

AI 기반 데이터 모델 개선

추진 배경 및 문제

사업건전성 대시보드에 추가 비즈니스 요구사항을 반영하기 위해 기존 데이터 모델의 개선이 필요한 상황이다. 현재 이를 위해서는 DB 전문가 2명을 2~3개월간 투입해야 하는 리소스가 요구된다. AI를 활용해 데이터 모델 개선 프로세스의 기간과 인력을 단축하고자 한다.

목표

AI 기반 데이터 모델 개선 검증 프로젝트를 추진하여 비즈니스 요구사항 정의→원천 데이터 정보 정의→데이터 모델 설계(ERD, 테이블 정의)→SQL 쿼리 개발 및 검증 전 과정에서 AI 활용 가능성을 확인한다.

프로세스 변화
기존 프로세스
  1. 기획자가 비즈니스 요구사항 도출
  2. DB 개발자가 기술 요구사항 및 설계
  3. DB 개발자가 SQL 쿼리 개발 및 검증
변경 프로세스
  1. 기획자가 비즈니스 요구사항 도출
  2. AI가 기술 요구사항 정의 및 데이터 모델 설계 (DB 전문가 검증·수정)
  3. AI가 SQL 쿼리 개발 및 검증 (DB 전문가 검증·수정)
해결해야 할 기술 문제
📂

원천 데이터 정보 주입 방법

AI가 데이터 모델을 올바르게 설계하려면 기존 테이블 DDL, 컬럼 정의, 데이터 관계, 원천 데이터 구조를 사전에 정확히 파악하고 있어야 한다. 이 정보를 AIPro에 어떤 방식으로 전달할 것인지(파일 업로드, 텍스트 직접 입력 등) 방법을 확정해야 한다.

확인 필요: AIPro의 파일 업로드 지원 형식 및 용량 한계
🧩

비즈니스 로직 정밀도 확보

사업건전성 대시보드는 BO 기준·솔루션 기준 등 복잡한 집계 조건이 존재한다. AI가 이러한 집계 기준을 혼동하거나 잘못 반영하는 환각(hallucination) 오류 발생 가능성이 높으며, 이를 탐지하고 수정하는 검증 절차를 어떻게 설계할지 결정해야 한다.

확인 필요: 복잡한 비즈니스 로직을 AI에게 전달하는 프롬프트 설계 방식
🗄️

SQL 실행 검증 환경

AI가 작성한 SQL 쿼리가 실제로 올바르게 동작하는지 검증하려면 실 DB 또는 테스트 DB에서 실행해볼 수 있어야 한다. AIPro가 DB에 직접 연결하여 쿼리를 실행·결과를 확인하는 기능을 지원하는지, 아니면 사람이 별도로 실행해야 하는지 확인이 필요하다.

확인 필요: AIPro 내 SQL 실행 및 결과 확인 기능 지원 여부
📐

ERD 산출물 형태

AI가 설계한 데이터 모델을 ERD 형태로 시각화하여 팀 내 공유 및 검토에 활용하려면, AIPro가 ERD를 이미지 또는 표준 형식으로 출력할 수 있어야 한다. 출력 형태와 활용 방법을 사전에 확인해야 한다.

확인 필요: AIPro의 다이어그램·ERD 출력 지원 여부
TASK 03

AI 기반 시안 생성 + 데이터 시뮬레이션 검증

추진 배경 및 문제

기존 대시보드 구축 시 시안 및 와이어프레임은 디자이너 중심으로 제작되어, 실제 데이터 구조 및 KPI 계산 로직이 설계 단계에서 충분히 검증되지 못하는 한계가 있었다. 이로 인해 데이터 반영 이후 지표 오류 및 시각화 부적합 발견, BO 기준·솔루션 기준 등 집계 기준 불일치, 개발 이후 반복 수정 및 리워크가 증가하는 문제가 발생했다.

목표

AI 기반 시안 생성과 데이터 시뮬레이션을 결합하여, 설계 단계에서 KPI 정합성과 화면 사용성을 사전 검증하는 체계를 마련한다. "개발 이후 검증"에서 "설계 단계 검증"으로 프로세스를 전환한다.

주요 기능
  • KPI 구조 및 화면 레이아웃 자동 설계 (요약 → 상세 → 원인 분석 흐름 구성)
  • 실제·가상 데이터를 적용한 지표 산출 로직 검증 (BO 기준 vs 솔루션 기준 등)
  • 시각화 적합성 및 가독성 검증
  • 지표 정의와 화면 구조 간 불일치 사전 제거
프로세스 변화
기존 프로세스
  1. 요구사항 정의
  2. 디자이너 시안 제작
  3. 대시보드 개발
  4. 실제 데이터 반영
  5. 지표 오류·시각화 문제 발견
  6. 수정 및 재개발
  7. 사용자 분석 및 의사결정
변경 프로세스
  1. 요구사항 정의
  2. AI 기반 시안 생성
  3. 데이터 시뮬레이션 수행
  4. KPI·시각화·UX 사전 검증
  5. 대시보드 개발
해결해야 할 기술 문제
🖼️

시안 렌더링 가능 여부

AI가 생성한 화면 시안(HTML 등)을 실제로 눈으로 확인하려면 렌더링 환경이 필요하다. AIPro 내에서 시각적 결과물을 미리보기로 제공하는지, 아니면 외부 도구에 붙여넣어야 하는지 확인이 필요하다.

확인 필요: AIPro의 HTML·화면 렌더링 미리보기 지원 여부
📊

데이터 시뮬레이션 실행 환경

가상 또는 실제 데이터를 입력하여 AI가 KPI 산출 로직을 실제로 계산하고 결과를 보여주는 기능이 필요하다. 단순 로직 설명에 그치지 않고 수치 계산 결과까지 확인할 수 있어야 한다.

확인 필요: AIPro의 데이터 계산·시뮬레이션 실행 기능 지원 여부
🔍

정합성 자동 검출 범위

지표 정의서와 화면 구조 간 불일치를 AI가 자동으로 탐지할 수 있는지, 아니면 사람이 AI의 도움을 받아 검토하는 방식인지 자동화 범위를 결정해야 한다. 자동화 수준에 따라 구현 방식이 달라진다.

확인 필요: AI 자동 정합성 검출의 정확도 및 신뢰 수준
TASK 04

대시보드 자동 분석 + 인사이트 리포트 메일 푸시

추진 배경 및 문제

현재 사업건전성 대시보드는 Tableau 기반으로 데이터 조회 및 시각화 중심으로 활용되고 있으나 다음과 같은 한계가 있다.

  • 접근성 한계: 사용자가 직접 대시보드에 접속해야 확인 가능하며, 지표 변화가 발생해도 즉시 인지하기 어렵다
  • 수동적 분석 구조: 이상 발생 시 사용자가 직접 원인을 분석해야 한다
  • 리포트 작성 부담: 분석 결과를 별도로 정리하여 보고해야 하는 수작업이 필요하다
목표

AI가 주요 KPI의 변화를 자동으로 감지하고 원인을 분석하여 인사이트 리포트를 생성, 담당자에게 메일로 자동 발송하는 체계를 구축한다. 사용자가 대시보드에 접속하지 않아도 주요 변화를 즉시 인지하고 의사결정에 활용할 수 있게 한다.

주요 기능
  • 수주율·실주율 등 주요 지표 변동 자동 감지
  • 솔루션·법인·기간별 영향도 분석 및 주요 변화 요인 도출
  • 자연어 기반 인사이트 리포트 자동 생성
  • 지표 이상 발생 시 인사이트 리포트 포함하여 메일 자동 발송
  • 자연어 질문 기반 추가 분석 (사용자 질의 대응)
프로세스 변화
기존 프로세스
  1. 사용자가 대시보드 직접 접속
  2. 데이터 확인
  3. 이상 인지
  4. 추가 분석 수행
  5. 리포트 수동 작성
  6. 보고 (접근성 낮음, 분석·보고 모두 수동)
변경 프로세스
  1. AI가 KPI 변화 자동 감지
  2. AI가 인사이트 자동 생성 (담당자 확인 및 검증 후 조정 필요)
  3. 리포트를 메일로 자동 푸시
  4. 사용자가 즉시 확인 및 의사결정
데이터 수집 방식 — 두 가지 접근 검토 필요

AI가 분석에 사용할 데이터를 가져오는 방식은 아래 두 가지를 모두 검토해야 하며, AIPro 환경과 사내 보안 정책에 따라 채택 방식을 결정해야 한다.

방법 A — DB 직접 읽기

AIPro가 사내 DB에 직접 연결하여 데이터 쿼리

  • Tableau 라이선스·API 없이 독립적으로 구현 가능
  • 원하는 집계 단위·기간·조건으로 자유롭게 추출 가능
  • 수치 정확도 높음, 리포트 품질 우수
  • 주기적 자동 실행 시 DB 부하 관리 필요
권장 방식
방법 B — 대시보드 읽기

Tableau REST API 또는 이미지(Vision) 방식으로 대시보드 데이터 추출

  • 기존 Tableau 자산 재활용 가능
  • REST API: 데이터 추출 정확하나 라이선스·권한 설정 필요
  • Vision 방식: 구현 빠르지만 수치 오류 위험 있음
  • Tableau 환경에 의존성이 남음
