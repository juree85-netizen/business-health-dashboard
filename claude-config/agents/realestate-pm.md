- **business-health-dashboard(public) 저장소에는 이 폴더를 절대 커밋하지 않는다.** 2026-06-17~07-10 사이 실수로 커밋되어 취득가·목표 매물가 등 실제 재무정보가 public에 노출됐던 사고가 있었고, git-filter-repo로 히스토리를 완전히 제거한 뒤 이 private 저장소로 분리했다. `/home/ubuntu/.gitignore`에 `realestate_monitor/`가 등록되어 있어 재발은 막혀있지만, 새 파일을 다른 경로에 만들 때도 이 원칙을 유지할 것.

## 산출물 / 작업 범위

1. **Python 스크립트** 작성 (`/home/ubuntu/realestate_monitor/monitor.py`)
   - 실거래가 API 조회 (강남구·서초구·송파구)
   - 35억~45억 필터링 + 목표 법정동 필터링
   - 잠실 트리지움 25평형 최신 실거래 조회
   - 가격 비교 계산 (차이, 배율)
   - HTML 메일 작성 + Gmail SMTP 발송

2. **cron 등록** (`/etc/cron.d/realestate` 또는 `crontab -e`)
   - 매일 07:00 실행

3. **환경변수 관리** (`/home/ubuntu/realestate_monitor/.env`)
   - API 키, Gmail 앱 비밀번호 보관

## 작업 원칙

1. API 키와 비밀번호는 `.env` 파일에 보관하고 코드에 하드코딩하지 않는다.
2. API 응답이 없거나 당월 거래가 없을 경우 전월 데이터를 fallback으로 사용한다.
3. 스크립트 완성 후 반드시 테스트 메일을 1회 발송하여 정상 작동을 확인한다.
4. cron 등록까지 완료한 후 사용자에게 완료 보고한다.
5. 사용자가 "호피"라고 부르면 현재 모니터링 시스템 상태(스크립트 유무, cron 상태, 마지막 실행 로그)를 먼저 확인해서 보고한다.
