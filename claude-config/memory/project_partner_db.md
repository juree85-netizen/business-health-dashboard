## 과제 개요
- 주관: 파트너전략그룹 (별도 과제)
- 발표일: 2026-03-30
- 개발 일정: 2026년 5월 개발 착수, 상반기 완료 목표
- 시스템 오픈 전(상반기): 월별 엑셀 매뉴얼 관리
- 시스템 오픈 후(하반기): 파트너십 사업 관리 담당자가 월 단위 엑셀 업로드 방식

## 관리 데이터 대상 (파트너전략그룹 계획)
① Tech/ISV 파트너십 체결 현황 정보 (파트너십 유형, 계약일자, 등급 등)
② 매출 실적 상세 정보 (수주내역, 고객, mySales 정보)
③ 파트너사로부터의 수혜 혜택·활용 정보 (펀드, 유상교육, 바우처 등)

## 사업부장(송해구) 코멘트
- 외부 상품 매출도 시스템으로 관리 필요
- 파트너 마스터 정보(리셀러 마진, 파트너십 체결비용 등)는 사업부와 협의해서 확정
- 리셀러 마진 등은 전체 공개 금지 → 권한 관리 철저히 요청

## STAR 시스템 화면 구성 (현재)
① 파트너십 현황 모판: 구분·파트너사·계약체결일·유형·파트너십 만료여부·등급·사업범위·독점권·사업부·담당부서
② 매출/사업 등록·관리: 구분·파트너사·대내외·사업부·실적년월·사업기회명·mySales·리빌링/라이선스금액·그 외 금액·합계
③ 수혜 혜택: 구분·파트너사·수혜년월·내용·금액·수혜부서

## 매출 필드 정의 확인
- 리빌링/라이선스 금액: 벤더에게 지불하는 라이선스 사용료 (= 외부 상품 매출)
- 그 외 금액: 우리가 추가 구축/커스터마이징하는 비용 (= MSP 매출)
- 목적: 리빌링 vs MSP 비율로 수익성 분석
- → 외부 상품 매출은 이미 리빌링 항목으로 커버됨. 매출 화면 추가 요청 불필요

## 리셀러 마진 확인
- 리셀러 마진 = 할인율(%) 의미
- 파트너십 계약 문서에 표시되어 있어 대부분 알고 있음 → 데이터 수집 가능
- 파트너사마다 다르며, 고객에게 비공개가 원칙 → 권한 관리 필요
- 리빌링 금액에 이미 포함된 경우도 있어 파트너사별로 다름

## STAR 시스템 추가 요청사항 (파트너전략그룹 전달 예정)
| 화면 | 요청 내용 | 사유 |
|---|---|---|
| 파트너십 현황 | 리셀러 마진(할인율, %) 필드 추가 | 파트너십 문서 기반 수집 가능, 권한 보유자만 조회 |
| 파트너십 현황 | 파트너십 체결비용(원) 필드 추가 | 사업부장 코멘트, 권한 보유자만 조회 |
| 매출 화면 | 추가 요청 없음 | 리빌링/MSP 구분으로 이미 커버 |
| 권한 관리 | 리셀러 마진·파트너십 체결비용 조회 권한 별도 설정 | 사업부장 코멘트 |

## 추가 제안 지표 (사업건전성 관점)
- 등급 유지 요건 vs 실적 달성률 (Revenue Target): 우선순위 높음
- 자격증/인증 현황 (요구 수 vs 보유 수 Gap): 우선순위 높음
- 수혜 혜택 활용률 (펀드·바우처 수령 대비 실사용): 우선순위 중간
- 파트너사 소개 리드(Lead) 현황: 우선순위 중간, 수집 난이도 높음
- 공동 마케팅·활동 이력: 우선순위 낮음

## 사업건전성 대시보드와의 연계
- 파트너전략그룹 별도 과제이나, 해당 DB를 사업건전성 대시보드에서 인터페이스하여 활용 예정
- DB 설계 단계에 사업건전성 대시보드에서 필요한 필드를 미리 반영 요청할 예정

**Why:** DB 구축 이후 필드 추가는 개발 공수 발생 → 설계 단계에서 선제적으로 요청해야 함
**How to apply:** 파트너전략그룹에 전달할 필드 요청 목록 작성 시 사업건전성 대시보드 지표와 연계하여 검토
