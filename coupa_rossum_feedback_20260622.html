1. 재무 분석 보완

FCF 흑자/적자 여부만 언급하는 것이 아니라, FCF 산식과 이자상환능력의 관계를 정확히 이해하고 설명할 수 있어야 한다는 지적.

□ 정리 필요 항목
  • FCF 정의 및 산식: OCF − CAPEX
  • 순이익(Net Income) → EBITDA → OCF → FCF 흐름 (감가상각·SBC 처리 포함)
  • FCF가 이자비용 차감 전/후인지 명확히 구분
  • 비용 절감 효과가 이미 FCF에 반영된 것인지, 별도 효과인지 구분
  • LBO 인수 시 차입금 이자를 실제 감당 가능한지 수치로 검증
  • 에보코(Evoco) 경유 Coupa가 제공한 재무자료 기반으로 FCF·순이익·매출 수치 재검증 후 보고서 반영
□ 사업부장님 코멘트
"FCF가 174M이고 이자가 310M인데 왜 버틸 수 있는지 설명할 수 있어야 한다."
"FCF, EBITDA, 감가상각비, SBC 관계를 이해하지 못하면 M&A 타겟 평가가 어렵다."
"비용 절감으로 200M을 아꼈다는 논리가 FCF에 이미 반영된 것인지 구분해야 한다."
FCF 산식 퍼널 도식화 (매출 → 영업이익 → EBITDA → OCF → FCF) + 한 팀장에게 에보코 자료 수령 → 수치 보정 후 재보고
2. Coupa vs. Medallia 비교 관점 보완

현재 보고서는 재무지표 중심이나, 실제 핵심 차이는 사업 구조에 있다는 의견.

구분 Coupa Medallia
매출 구조 S2P/BSM 기반 반복 매출, 고객 Lock-in 高 CX·서베이 중심 기능형 SaaS
시장 지속성 조달·지불 프로세스는 사라지지 않는 시장 대체 서비스 전환 장벽 낮음
AI 대체 가능성 워크플로우·데이터·네트워크 결합 플랫폼 → 低 단기능 SaaS → 高
인수 당시 FCF 흑자 (+174M) + 비용 절감 여지 충분 적자 or 이자 충당 불가 수준
결과 비용 절감 → 이자 상환 → Value-up 성공 이자 부담 가중 → 투자금 전액 손실

핵심 메시지 — M&A 평가 시 재무지표뿐 아니라 반복 매출 안정성 / 시장 지속성 / 플랫폼 고착도 / AI 대체 가능성을 함께 봐야 함.

비교표에 인수 당시 매출 항목 추가 + 사업구조 관점 시사점 보완

3. SaaS/AI M&A 검토 체크리스트 작성

이번 보고서의 최종 산출물은 Coupa 사례가 아니라 "사업부 관점에서 SaaS/AI 기업 M&A 시 무엇을 봐야 하는가"를 정리한 체크리스트여야 한다는 지시.

□ Financial Metrics
  • 매출 성장률 / ARR 성장률
  • Gross Margin / EBITDA / EBITDA Margin
  • FCF / FCF Margin
  • Rule of 40 (성장률 + 이익률 ≥ 40 기준)
  • 이자상환능력 (FCF ÷ 연이자비용)
  • 현금보유액 / 부채 규모 / 차입금리
□ Operational Metrics
  • Churn Rate / NRR(Net Revenue Retention) / GRR(Gross Revenue Retention)
  • 고객 유지율 / 고객 집중도
  • CAC Payback / LTV:CAC
□ Business Quality
  • 반복 매출 안정성
  • 시장 지속성 (수요가 사라질 수 있는 시장인가)
  • AI 대체 가능성
  • 업무 프로세스 고착도 / 데이터 경쟁력 / 네트워크 효과
  • 경쟁사 대비 차별성
□ M&A 관점
  • 인수 후 비용 절감 가능성 및 규모
  • 통합(PMI) 난이도
  • 기존 솔루션과 시너지
  • Tuck-in 적합성
  • 실사(Due Diligence) 시 추가 검증 항목
□ 사업부장님 코멘트
"기획팀이 해주는 것을 기다리지 말고 사업팀도 스스로 판단할 수 있어야 한다."
"나중에 M&A 후보를 골라보자 할 때 사업팀 입장에서 무엇을 봐야 하는지 설명할 수 있어야 한다."
GPT 등 활용해서 지표별 기준치 스터디 → 체크리스트 초안 작성 → 지원팀장과 검토 → 확정
4. Rossum IDP 기술 검증

Rossum 자료는 공급사 주장 중심이므로 객관적 검증이 필요하다는 의견.

□ 검증 필요 항목
  • Rossum Aurora가 실제 LLM인지, OCR + ML 고도화인지 여부
  • SAP Ariba의 IDP 기능 수준 (Ariba에 IDP가 없다고 단정하면 안 됨)
  • UiPath·ABBYY 등 경쟁사와의 실질적 차별성
  • Coupa가 선제 확보한 것인지, 경쟁사 대비 따라잡기 성격인지
□ 사업부장님 코멘트
"공급사 주장을 그대로 적으면 안 된다. 항상 경쟁사와 비교해서 검증해야 한다."
"Ariba에 IDP가 정말 없는지 확인해야 한다."
"Coupa는 원래 IDP를 서드파티로 쓰세요 전략이었다. 따라잡기일 수 있다."
SAP Ariba / UiPath / ABBYY IDP 현황 조사 → 경쟁사 비교표 보완

5. 물류·SRM 의견 수렴 추진

Rossum IDP는 물류 영역과 직접 관련성이 높으므로 전문 조직 검토를 먼저 수행.

□ 공유 자료 발췌 기준
  • 포함 : Rossum 기업 개요 + Aurora(T-LLM) 기능 + 데모 영상
  • 제외 : Coupa M&A 내용, 쿠파 재무 자료 전체
□ 공유 순서 (순서 준수 필수)
1단계 안대영 상무
(물류사업부)
→
의견 수령 후 검토 결과 취합
→
2단계 권영웅 팀장
(SRM)
□ 1단계 — 안대영 상무 (물류사업부, IDP 담당)
  • 전달 맥락 : "물류 차원에서 검토에 도움이 될 것 같아서 한번 보시라"
  • 요청 사항
    • 물류 관점에서 해당 기능이 유의미한지
    • 현재 검토 중인 IDP 기술 대비 수준은 어떤지
    • 실제 적용 가치가 있는지
□ 2단계 — 권영웅 팀장 (SRM)
  • 안대영 상무 의견 수렴 후 공유
  • 확인 사항 : SRM 적용 가능성 / 유사 기능 개발 계획 / 외부 솔루션 활용 계획
로썸 자료 발췌 → 안대영 상무 전달 → 의견 수령 → 권영웅 팀장 공유 (순서 준수)
6. M&A 역량 학습 체계 구축

이번 과제를 단순 보고서가 아닌, M&A 판단 역량을 키우기 위한 학습 과정으로 활용할 것을 요청.

□ 정리 필요 개념
  • FCF / EBITDA / SBC / CAPEX 산식과 관계
  • Rule of 40 의미 및 SaaS 우량 기업 판단 기준
  • LBO 구조 이해 (부채 비율, 이자상환능력, Exit 전략)
  • M&A 실사(Due Diligence) 시 사업부 관점 체크포인트
  • ARR 성장률·Churn 기준치 ("이만큼 이상 나와줘야 된다" 수준으로 구체화)
