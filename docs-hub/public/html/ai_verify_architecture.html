VIEW 2WBS / 로드맵

리스크 낮은 순 착수(Task 3→2→1→4)에 완료정의(DoD) 오너 의존성 버퍼 성공 기준 Task 5 운영전환을 추가한 실행 가능 버전.

PHASE 0 도구 검증 스파이크 (2주) — 전체 선행 조건. VDI·GitHub(code.sdsdev.co.kr) 연결 확인 · AIPro/Claude 기본 동작 · 데이터 등급표 확정 · AIPro 기능 범위 정의. 실패 시 Plan B 채널 결정.
순서Task기간오너의존성완료 정의 (DoD)
1 Task 3
시안 + 시뮬레이션		 Wk 3–5 동료 B (프론트) Phase 0 대표 KPI 1~2개 화면 시안·프로토타입 완성 · 마스킹/합성 데이터로 동작 · UX 검토 통과. 리스크 최저, 데이터 의존 낮음. Task 2와 병렬 착수.
2 Task 2
데이터 모델		 Wk 3–6 동료 A (데이터) Phase 0 SQL·ERD 확정 · Impala 검증으로 정합도 ±5% 이내 입증 · 시안 지표와 매핑 완료. 담당자·의존성 다름 → Task 3와 동시 착수.
3 Task 1
웹 재구축		 Wk 6–10 동료 C (PM) Task 3 + Task 2 Web/WAS 실제 구현 · 시안+데이터 모델 통합 · 지표 10~15개 재현 · 기존 Tableau 화면 대비 완성도 평가.
4 Task 4
자동 리포트		 Wk 10–12 동료 C + 동료 A Task 1 월별 리포트 자동/반자동 생성 · 앞선 3개 산출물 기반 · 최종 검증 로그 정리.
5 Task 5 신규
운영 전환 (조건부)		 Wk 12~ +버퍼 동료 C Task 1·4 호스팅·인증·배포·유지보수 주체 결정 · 운영 전환 가능 여부(Y/N + 조건) 판정. 가설의 "운영·유지보수" 검증 대응.
🟢[검증 반영 · 신규] Task2·3는 담당자·의존관계가 독립적이므로 순차가 아닌 병렬 착수 권고. 확보되는 약 2주는 그대로 프로젝트 버퍼로 전환.
⏱[검증 반영 · 수정] 7/3 킥오프 기준 Phase0 2주 + Wk3~14 종료 시점은 10/9 전후. 마감이 '10월 말'이 맞다면 약 3주 버퍼 확보 가능 — 단, 정확한 마감일을 킥오프에서 먼저 확정할 것. 마감일이 10월 초·중순이라면 버퍼는 그만큼 줄어드니 재계산 필요.

■ 검증 성공 기준 (Exit Criteria) — 원본 미정의분, 킥오프에서 확정 필요

1데이터 정합도 ≥ 95% (기존 Tableau 대비 지표값 일치)
2공수·리드타임 절감률 — 기존 전문가 방식 대비 man-day / 기간
3화면 재현율 — 대상 지표 N개 중 재현 성공 개수
4운영 전환 가능 여부 — Y/N + 전환 조건 명시
5[검증 반영 · 신규] 이해관계자 만족도 — 실사용 부서(사업부장 등) 대상 화면 품질·사용성 정성 평가(5점 척도 등), 정량 지표 통과와 별개로 확인. 정량 재현율은 높아도 실무 적용 가능성이 낮으면 Task 5(운영전환) 판단에 반영.
※ 비교 기준선(Baseline) 선확보: "전문가 없이 가능한가"를 정량 입증하려면 기존 Tableau 방식의 공수·기간·품질을 대조군으로 먼저 측정해야 함.

VIEW 4거버넌스 & 역할 구조

3인 R&R · 의사결정 흐름 · bus factor 대비 · 데이터 반출 승인. 역할 구성은 제안안이며 킥오프에서 협의·확정.

■ 의사결정 흐름 & 킥오프 확정 대상

1역할 분담 최종 확정 — 제안안 → 협의 확정 (오너·기한 부여)
2검증 범위 — 상품 1~2개 · 지표 10~15개 · 운영 포함 여부
3VDI 방화벽 GitHub 접근 확인 (Phase 0 필수)
4AIPro 기능 범위·버전 정의 (Phase 0로 승격)
5컨텍스트 공유 방식 결정 (릴레이 or Plan B)
6데이터 반출 승인 절차 신규 — 누가 무엇을 승인하는가
7성공 기준(Exit Criteria) 신규 — VIEW 2 정량 지표 확정
8[검증 반영 · 신규] 동료 A·B의 실제 SQL/디자인 경력 수준 확인 및 문서화 — 가설이 검증하려는 건 '비전문가 + AI'이므로, 두 분이 이미 전문가 수준 배경이 있다면 가설 자체를 '준전문가 + AI 도구'로 재정의할 것.
※ 결정사항 각 건에 오너·기한(due date)을 붙여 킥오프 아웃풋 = "누가 언제까지"로 관리. 1인 이탈 대비(bus factor)를 킥오프에서 한 줄이라도 합의할 것.
