## WHY — 왜 만드는가

### 1. 배경 (현재 상태의 문제)

| 축 | 문제 |
|----|------|
| 지표 | 매출/영업이익 중심 관리 → 장기 사업건전성 판단 어려움 |
| 데이터 | 부서·시스템별 분산, 수작업(엑셀/워드) 보고 → 현황 파악 지연 및 트렌드 분석 한계 |
| 관리체계 | SI 중심 운영 경험 → SaaS 사업관리 역량 부족, 대외 사업 확장 대응 선진체계 필요 |

### 2. 문제 정의

> 사업부장·팀장이 솔루션 사업의 건전성을 **종합적·장기적으로 파악**하고 **전략적 의사결정**을 수행할 수 있는 **지표·체계·도구가 부재**함

---

## WHAT — 무엇을 만드는가

### 3. 목적

> 단기 성과와 장기 건전성을 **통합적으로 파악**하고, 이슈·위험을 **조기 감지**하여 **데이터 기반 의사결정**을 지원하는 사업건전성 대시보드 구축

### 4. 목표 (3가지)

| # | 목표 |
|---|------|
| ① | 통합 사업건전성 분석 체계 구축 |
| ② | 경영관리 의사결정 지원 체계 확립 |
| ③ | 성장단계 기반 KPI 관리체계 수립 |

---

## HOW — 어떻게 만드는가

### 5. 추진방안 3축

#### ① 분석 — 다차원 통합 분석

- **분석 영역**: 재무(성장/수익/효율), 고객(고객상태별 현황·고객만족·VoC·MAU), 시장, 영업 — 영역별 + 종합 현황
- **핵심 지표**: ARR · ACV · Recurring · Rule of 40 · Gross Margin · GSE · NSE · 고객상태별 매출 · 고객 수
- **분석 차원**: 솔루션별 / 매출유형(ARR·Recurring·Device 등) / 고객상태(New·Expansion·Contraction·Churn) / 법인별
- **분석 주기**: 월 / 분기 / 반기 / 연 / YTM
- **비교 기준**: 실적 vs. 전년·계획 / 전망 vs. 전년·계획 / 실적 vs. 직전 실적
- 매출·원가 구조 분석 포함

#### ② 프로세스 — 경영관리 사이클 연계

- 월/분기/반기 실적 및 전망 관리 (중복 시 장기 기준 우선 — 예: 4월은 1분기 실적만)

| 시점 | 내용 |
|------|------|
| 6월 | 상반기 평가 (1~5월 실적 + 6월 전망) / 상품 중장기 로드맵 수립 |
| 9월 | 사업 중장기 로드맵 수립 |
| 11월 | 연간 평가 (1~10월 실적 + 12월 전망) |
| 9~12월 | 차년도 경영계획 수립 (총 3차 수정·확정) |

#### ③ KPI — 성장단계별 체계 적용

- 성장단계별 KPI 차등 적용: **도입 → 성장 → 성숙 → 쇠퇴**
- 상품 특성별 KPI 별도 설계
- 단계별 사업건전성 판단 기준 정립
