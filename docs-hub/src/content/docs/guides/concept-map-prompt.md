### STEP 1 — 사전 지식 점검
대상에 대해 충분히 알고 있는지 먼저 판단한다.

- **잘 아는 경우** → 바로 개념도 작성
- **모르거나 불확실한 경우** → 웹 검색 먼저 실행

`WebSearch` 도구로 대상을 검색해 팩트를 확보한 후 개념도 작성.

## 개체 분류 방식 (v3)

고정 목록 없음. AI가 대상의 본질을 가장 잘 표현하는 말로 자유롭게 기술.

```
두쫀쿠 → 음식 · 한국 트렌드 디저트
피터 드러커 → 인물 · 경영 사상가
행동경제학 → 학문 · 경제학·심리학 교차 분야
맥킨지 7단계 → 프레임워크 · 경영 문제해결 방법론
첼로 → 악기 · 현악기
```

복합적인 대상은 복수 표기 가능.

## 출력 구조 — 0 + 8개 항목

| # | 항목 | 역할 | 섹션명 예시 |
|---|------|------|------------|
| 0 | **개체 분류** | 자유 분류 + 한 줄 요약 | — |
| 1 | **상위 맥락** | 대상이 속한 더 큰 범주 | 상위 개념 / 시대적 배경 / 음악 계통 |
| 2 | **등장 배경** | 왜 존재하는가 | 존재 목적 / 발명 경위 / 주요 업적 |
| 3 | **동일 층위 비교** (MECE) | 같은 레벨 비교 | 유사 음식 / 동시대 인물 / 경쟁 제품 |
| 4 | **대비·영향 관계** (MECE) | 대비·경쟁·영향 | 대체재 / 영향 관계 / 반대 개념 |
| 5 | **하위 구성요소** (MECE) | 내부 분해 | 재료·조리법 / 주요 저작 / 하위 분야 |
| 6 | **실제 사례** | 구체적 적용 맥락 | 대표 브랜드 / 대표 작품 / 적용 사례 |
| 7 | **트리 구조** | 계층 시각화 | `├──` 형식 고정 |
| 8 | **비유** | 일상 언어 착지 | — |

## 핵심 원칙

| 원칙 | 내용 |
|------|------|
| 검색 우선 | 모르는 대상은 웹 검색으로 팩트 확보 후 실행 |
| 자유 분류 | 고정 목록 없이 대상 본질에 맞게 서술 |
| 지형도 우선 | 정의보다 "어디에 속하는가"를 먼저 설명 |
| 섹션 어댑테이션 | 섹션명은 대상 성격에 맞게 자동 조정 |
| MECE 분류 | 3·4·5번 항목은 겹치지 않고 빠짐없이 구성 |
| 비유 포함 | 추상적 개념을 일상 언어로 착지 |
