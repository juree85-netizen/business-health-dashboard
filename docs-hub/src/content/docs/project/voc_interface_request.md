# VoC 데이터 인터페이스 요청서
#### (기존 solution_if_voc 유지 + 신규 테이블 및 API 추가)

---

## 0. 배경

- 기존 `solution_if_voc` 테이블은 DATALAKE API(`/datalake/QualitySummary`)를 통해 VOC건수·불만건수·실사용자수 3개 지표를 월별 집계(CNT) 형태로 수신 중 (배치 주기: 1일 1회)
- 기존 데이터는 월별 합산값(PROD3_NM / GUBUN / YYYY / MM / CNT 구조)이므로, 개별 건 단위 분석(리드타임·지연 여부 등) 불가
- 신규 지표(접수월별 개선건수·평균 리드타임·지연건수) 산출을 위해 CS 대시보드로부터 건별 데이터 추가 필요
- **인터페이스 방식 확정: 신규 테이블(건별 데이터) 방식 채택**
  - 기존 테이블(`solution_if_voc`): 현행 유지
  - 신규 테이블: CS 대시보드 건별 데이터 적재 → 솔루션사업부가 직접 집계 및 분석
  - 기존 창구(`/datalake/QualitySummary`)는 그대로 유지, 신규 데이터 제공 창구(API) 추가 개발 요청
  - 기존 파라미터의 조회 범위 1년 이내 제한 해제 필요 (당사 요청 범위: 2023년 1월~현재, 약 3년치)
- CS 대시보드는 현재 시점 스냅샷 방식 운영 중. 당사는 시계열 트렌드 분석 목적이므로 매월 1회 제공 요청

> **※ CS 대시보드 누계지연건수를 사용하지 않는 이유**
> CS 대시보드에서 제공하는 누계지연건수는 현재 시점의 미완료 건만 집계하여 완료 처리된 지연건이 누락됩니다. 정확한 월별 지연건수 추이 관리를 위해 CS 대시보드 건별 데이터 수신 후 솔루션사업부가 직접 산출합니다.

### 테이블 구조 개요

| # | 테이블 | 역할 |
|---|--------|------|
| 1 | `solution_if_voc` | 기존 월별 집계 (현행 유지) |
| 2 | `solution_if_voc_detail` | CS 대시보드로부터 건별 데이터 16개 필드 (신규 인터페이스 요청) |
| 3 | `sol_if_voc_snap` | 리드타임·지연건수 월별 스냅샷 — 솔사 내부 개발 예정 **(본 인터페이스 개발 범위 아님)** |

## 6. 산출 예정 지표

수신된 CS 대시보드 건별 데이터 및 스냅샷 테이블을 활용하여 아래 3개 지표 산출

| # | 지표명 | 계산식 | 활용 테이블/필드 | 스냅샷 필요 |
|---|--------|--------|-----------------|------------|
| 1 | 접수월별 개선 요청 건수 | 요구유형 = '개선'인 건수 (기준월별, 상품별) | 기준월, 상품/서비스, 요구유형 | ✕ |
| 2 | 접수월별 평균 처리 리드타임 | SUM(리드타임(시간)) ÷ COUNT(진행단계='완료') | 기준월, 상품/서비스, 진행단계, 리드타임(시간) | ✓ |
| 3 | 월별 지연 건수 | 아래 산출식 참조 | 예정일자, 완료일자, 상품/서비스 | ✓ |

**※ 평균 리드타임 산출식**

```
평균 리드타임(M월) =
  SUM(처리 완료시점 − VoC 접수시점)
  ÷ COUNT(VOC_ID)
  WHERE 기준월 = M
    AND 진행단계 = '완료'
    AND 상품/서비스 = [상품명]
```

> ※ M월에 접수된 건 중 완료 처리된 건만 대상으로 산출. 이후 월에 완료되는 건이 추가될수록 값이 변하므로 스냅샷 저장 필요.

**※ 지연건수 산출식**

```
지연건수(M월) = COUNT(VOC_ID)
  WHERE 상품/서비스 = [상품명]
    AND (
      완료일자 > 예정일자                              -- 완료됐으나 기한 초과한 건
      OR (완료일자 IS NULL AND 예정일자 < M월 말일)    -- 미완료이며 기한이 지난 건
    )
```

> ※ CS 대시보드의 누계지연건수는 현재 시점 미완료 건만 집계하여 기완료 지연건이 누락됨. 위 산출식을 통해 완료 여부와 무관하게 기한 초과 건 전수를 정확히 산출함.

## 7. [참고] 내부 스냅샷 관리 (`sol_if_voc_snap`)

> ※ 본 항목은 CS측 인터페이스 요청 사항이 아님. 솔루션사업부에서 수신된 건별 데이터를 활용하여 내부적으로 운영하는 테이블이며, **CS측 개발 범위에 미포함.**

리드타임·지연건수는 계산 시점에 따라 값이 달라지는 지표이므로, 매월 고정 시점에 계산한 값을 솔루션사업부 내부 테이블에 저장하여 일관된 추이를 관리합니다.

- **개선 요청 건수**: 접수 시점 값이 고정되므로 스냅샷 불필요, 건별 데이터에서 상시 재계산
- **리드타임**: 완료건 수가 매월 누적되며 평균값이 변하므로 월별 고정 시점 저장 필요
- **지연건수**: 미완료건 상태 변화에 따라 값이 달라지므로 월별 고정 시점 저장 필요

**스냅샷 테이블 필드 (`sol_if_voc_snap`)**

| # | 필드명 | 설명 |
|---|--------|------|
| 1 | 기준월 | 지표 산출 기준 월 (예: 2026-03) |
| 2 | 스냅샷 일자 | 실제 계산·저장 일자 (예: 2026-04-01) |
| 3 | 상품/서비스 | 대상 상품명 |
| 4 | 평균 리드타임(시간) | 해당 기준월 완료건 기준 평균 처리 시간 |
| 5 | 지연건수 | 해당 기준월 기준 지연 건수 |

**적재 방식**

| # | 항목 | 내용 |
|---|------|------|
| 1 | 산출 시점 | 매월 1일 (전월 기준 계산) |
| 2 | 적재 방식 | M-1월 기준 계산값을 1행씩 추가 적재 (기존 누적 데이터 유지) |
| 3 | 관리 주체 | 솔루션사업부 내부 개발 예정 (CS 개발 범위 아님) |
