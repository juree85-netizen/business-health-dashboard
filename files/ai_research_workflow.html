1단계 · 목적 정의

가장 중요한 단계입니다. 목적이 흐릿하면 AI도 방향을 잡지 못합니다. 시작 전에 아래 3가지를 한 문장씩 확정하세요.

🎯
조사 목적
이 조사로 무엇을 결정하거나 이해하려는가?
예: PE 기반 SaaS 인수 실패 원인 학습
👥
보고 대상
누가 읽는가? 팀장/사업부장/팀 전체?
예: 그룹장 보고용 (1페이지 요약 필요)
📄
산출물 형태
분량, 형식, 제출 기한
예: A4 6장, Word 파일, 이번 주 금요일
좋은 목적 정의 예시:
"PE가 SaaS 기업을 인수할 때 어떤 기준으로 평가하는지, Medallia 실패 사례를 통해 분석하여 우리 솔루션 M&A 검토 시 적용 가능한 평가 기준을 도출하고 싶다."
2단계 · Input 수집

AI에게 줄 재료를 모읍니다. 재료가 풍부할수록 결과물의 품질이 높아집니다.

Input 종류

🔗
기사·링크
Bloomberg, Reuters, 업계 뉴스 기사 URL
IR 자료, 공시 문서 링크
분석 보고서 링크
💡
사람의 관점·가설
"이 사례에서 특히 ___가 궁금하다"
"우리 회사 관점에서 ___를 강조하고 싶다"
이미 알고 있는 배경 지식
📋
양식·제약 조건
사용할 보고서 양식 (sds-word-writer)
분량 제한
포함해야 할 필수 섹션

Input 품질 체크리스트

  • 링크가 실제로 열리는지 확인했는가?
  • 링크 외에 내가 직접 알고 있는 정보(회의 내용, 내부 자료)가 있으면 텍스트로 정리해서 붙여넣기
  • "이 정보는 ___에서 나온 것"이라고 출처를 함께 제공
  • 원하는 분석 각도(긍정/비판적/중립)를 명시
주의: 링크만 던지면 AI가 방향을 추측합니다. "이 링크에서 특히 자금 구조와 실패 원인 파트를 중심으로 분석해줘"처럼 집중 포인트를 함께 지정하면 정확도가 크게 올라갑니다.
3단계 · AI 리서치 (AI가 수행)

Input을 받은 AI가 수행하는 작업입니다. 사람은 기다리기만 하면 됩니다.

AI가 하는 것

🔍 링크 내용 분석
제공받은 URL의 전체 내용 파악, 핵심 수치·사실 추출
🌐 추가 웹 검색
링크에 없는 맥락 정보, 경쟁사 비교, 시장 데이터 검색
🗂 구조 설계
목차 구성, 섹션별 내용 배분, 분량 조절
✍ 초안 작성
보고서 문체로 전환, 표·그래프 데이터 정리, 별첨 구성

AI가 하지 않는 것 — 사람이 반드시 해야 할 것

📌 제목 선정
보고서 제목은 독자의 관심을 결정합니다. AI는 내용을 요약한 제목을 제안하지만, "이 보고를 왜 하는가"를 담은 제목은 사람만이 정할 수 있습니다.
📎 데이터 참고문헌의 명확성
AI가 인용하는 수치의 출처가 명확한지 확인하세요. "추정치", "Bloomberg 기반", "공시 기준" 등 데이터 신뢰도와 출처를 직접 검증하고 표기해야 합니다.
🤔 왜 저런 행동을 하지? — 동기·이해관계 해석
AI는 사실을 나열하지만, "누가 손해고 누가 이익인가", "왜 저 시점에 그 결정을 했나"는 업계 경험과 인간의 직관이 있어야 해석됩니다.
🔍 데이터가 없을 때 — 논리적 추론
공개 데이터가 없는 영역에서 AI는 추정을 거부하거나 일반론을 답합니다. 유사 사례 비교, 업계 평균 적용, 상황 추론은 사람의 판단이 필요합니다.
🏢 내부 비공개 정보
내부 회의 내용, 우리 회사만 아는 사업 맥락과 전략 방향. AI가 접근할 수 없으므로 직접 알려줘야 합니다.
👁 최종 수치 검증
AI 수치는 반드시 원본 대조 확인. 보고 받는 분의 관심 포인트 파악은 경험에서 나오는 판단입니다.
소요 시간: 기사 3~5개 + 웹 검색 기준 약 10~20분. 복잡한 재무 분석이 포함되면 30분까지 소요될 수 있습니다. AI가 작업 중이면 다른 업무를 병행해도 됩니다.
4단계 · 사람의 검토 & 통찰 주입

AI 초안을 보고서 완성본으로 만드는 핵심 단계입니다. AI가 채울 수 없는 부분을 사람이 채웁니다.

검토 체크리스트

  • 수치가 실제 출처와 일치하는지 확인 (AI는 추정치를 쓸 수 있음)
  • 맥락 오류: "이건 우리 업계에서 다르게 작동한다"는 부분 수정
  • 우리 회사 관점 추가: "이 사례가 우리 솔루션에 주는 시사점"
  • 강조점 조정: 보고 받는 분이 관심 가질 부분을 앞으로 이동
  • 분량 조절: 너무 길면 컷, 부족하면 보완 요청

수정 요청 방법

수정 요청 예시
[AI 에이전트명], 3장 시사점 섹션에서 "우리 솔루션의 Gen AI 대체 가능성" 파트를 좀 더 구체적으로 써줘. # 우리 회사 솔루션 중 Knox Suite는 데이터 폐쇄성이 높아서 대체 가능성이 낮다는 점을 넣고, # Brity Automation은 반대로 대체 위험이 있다는 걸 언급해줘. 그리고 전체 분량을 A4 6장에 맞게 조정해줘.
통찰 주입 팁: "이렇게 바꿔줘" 대신 "우리 업에서는 ___이다, 이 맥락을 반영해줘"처럼 배경 정보를 주면 AI가 더 정확하게 반영합니다.
5단계 · Word 문서 생성 (sds-word-writer)

내용이 확정되면 sds-word-writer 스킬을 사용해 보고용 .docx 파일을 자동 생성합니다.

Word 생성 트리거 방법

프롬프트 예시
[AI 에이전트명], 지금까지 정리한 내용으로 sds-word-writer 스킬을 써서 보고용 Word 파일 만들어줘. 파일명: medallia_lbo_analysis_20260527.docx 저장 위치: files/ # 문서 허브에도 링크 추가해줘.

Word 생성 시 자동 적용되는 것

항목 자동 적용 내용
문서 양식 SDS 표준 보고서 양식 (헤더·푸터·폰트·여백)
목차 섹션 구조 기반 자동 생성
표·목록 보고서 스타일로 자동 변환
파일 저장 /home/ubuntu/files/ 에 .docx 저장
문서 허브 등록 요청 시 HTML 문서 목록 페이지에 링크 추가
프롬프트 예시 모음

A. 기사 링크 기반 기업 조사 시작

처음 요청할 때
[AI 에이전트명], 기업 조사 부탁해. ## 목적 Qualtrics가 SAP에서 분리된 이후 어떻게 됐는지 알고 싶어. 우리 팀에 공유할 10분짜리 설명용 자료 만들 거야. ## 참고 링크 - https://... (Bloomberg 기사) - https://... (IR 자료) ## 특히 알고 싶은 것 - 분리 이후 재무 상황이 어떻게 달라졌는지 - Medallia와 어떻게 경쟁하고 있는지 ## 산출물 A4 5장 분량 보고서 초안 → 나중에 Word로 변환 예정

B. 내부 정보 추가 주입

사람의 통찰을 넣을 때
[AI 에이전트명], 시사점 섹션에 아래 내용을 반영해줘. # 우리 팀 내부 논의에서 나온 건데, 우리 솔루션 중 Brity Works는 워크플로우 자동화라서 GenAI 대체 위험이 높은 편이야. 이 관점을 2번 시사점에 추가해줘. 반면 Knox Suite는 보안 규제 데이터를 다루기 때문에 AI 대체 가능성이 낮다는 점도 균형 있게 써줘.

C. Word 파일 생성 요청

최종 Word 생성 시
[AI 에이전트명], 지금까지 정리한 내용으로 sds-word-writer 스킬로 Word 파일 만들어줘. 파일명: qualtrics_analysis_20260527.docx 저장: files/ 문서 허브 HTML 목록에도 추가해줘. 커밋·태그·푸시까지 해줘.
Tips & 주의사항
✅ 잘 되는 것
  • 공개된 재무 수치 정리·비교
  • 기사 여러 개를 합쳐서 흐름 파악
  • 표·도표 자동 구성
  • 보고서 문체로 변환
  • 별첨 목차 자동 생성
⚠ 주의할 것
  • 재무 수치는 반드시 원본 대조 확인
  • 최신 뉴스(1주 이내)는 검색이 느릴 수 있음
  • 비공개 정보는 사람이 직접 입력해야 함
  • AI 의견·예측은 참고용으로만 활용
중요 — AI 수치 검증 필수:
AI가 제시하는 재무 수치(특히 추정치·예상치)는 반드시 원본 자료와 대조하세요. Medallia 보고서에서도 "Bloomberg, OnlyCFO 분석 기반 추정치"임을 명시했듯이 출처가 불명확한 수치는 보고서에 "(추정)" 표기를 붙여야 합니다.
