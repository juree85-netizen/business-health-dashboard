수정된 전제

PHASE 0

도구 검증 스파이크 — 2주, 착수 전 필수 선행

목적

4개 과제 전체가 "AIPro/FabriX가 가능한지 확인 필요"라는 전제를 공유한다. 이 전제가 무너지면 프로젝트 방향 자체가 바뀐다. Phase 0은 2주 내 이 불확실성을 제거하고, 이후 과제 순서와 방식을 확정하기 위한 짧은 스파이크다.

검증 항목 4가지

① 대용량 컨텍스트 전달

지표 정의서 + DB 스키마 + 기존 Tableau 구조를 AIPro/FabriX에 파일 업로드 또는 텍스트로 입력 → AI가 정확히 참조하는지 확인

→ 결과: 지원 O/X, 용량 한계, 정확도

② SQL 생성 → 사람이 실행하는 루프

AIPro/FabriX가 SQL을 생성하면 담당자가 Windows PC Impala에서 실행 → 결과를 AI에게 피드백 → 수정 반복 가능한지 확인

→ 결과: 루프 가능 O/X, 맥락 유지 여부

③ HTML 시안 생성 + 외부 렌더링

AI가 생성한 HTML 코드를 브라우저 또는 문서허브(AWS)에 붙여넣어 시각적 결과 확인 → AIPro/FabriX 내 렌더링 없이도 검증 가능한지 확인

→ 결과: 외부 렌더링 대체 워크플로우 정의

④ 스케줄 자동화 가능 여부

AIPro/FabriX가 배치/스케줄 실행을 지원하는지 확인. 미지원 시 "사람이 주 1회 수동 실행 + 결과 메일 발송"으로 Task 4 범위를 재조정

과제 순서 (수정)

4개 과제 — 리스크 낮은 순으로 재편

기존 Task 1~4 번호 대신, 리스크와 의존성 기준으로 실행 순서를 재배치한다. 앞 단계 실패 시 뒷 단계 방식을 조정할 수 있도록 단계 간 체크포인트를 설정한다.

1st

Task 3 → AI 기반 시안 생성 + 데이터 시뮬레이션

DB 접근 불필요 가상 데이터 사용 가능 리스크 최저

가상 데이터로 KPI 산출 로직을 시뮬레이션하고, AI가 HTML 시안을 생성하여 문서허브에서 렌더링 확인. DB 접근 없이도 AI의 설계 능력과 팀의 검증 역량을 동시에 테스트할 수 있어 첫 번째 과제로 적합.

검증 성공 기준: AI가 생성한 시안이 기존 와이어프레임 스펙(6Row 12카드)을 충족하고, 가상 데이터 KPI 수치가 정의된 산식과 일치하면 성공
실패 시 대응: 시안 품질 미달 → 프롬프트 설계 방식 개선 후 재시도. 이 단계 실패는 Task 1(웹 재구축)의 방식도 수정 필요 신호
2nd

Task 2 → AI 기반 데이터 모델 개선

Impala DB 실행 필요 동료 A 주도

DB 스키마를 AI에게 제공하고 ERD·테이블 정의·SQL 쿼리를 생성하게 한 뒤 Impala에서 실행·검증. Task 1(웹 재구축)의 백엔드 SQL 레이어를 먼저 완성해야 화면에 실제 데이터를 연결할 수 있으므로 Task 1보다 선행.

검증 성공 기준: AI 생성 SQL이 Impala에서 오류 없이 실행되고, 기존 Tableau 수치와 ±5% 이내 정합성을 보이면 성공
실패 시 대응: 환각(hallucination) 오류 다발 → 비즈니스 로직을 더 구체적인 단위로 분할해 AI에게 전달하는 프롬프트 전략 재설계
3rd

Task 1 → AI 기반 웹 대시보드 재구축

Task 2 완료 후 착수 AWS 문서허브에 배포

Task 3에서 검증된 시안 + Task 2에서 완성된 SQL을 결합하여 실제 작동하는 웹 대시보드를 구축. 프론트엔드(HTML/JS)는 AWS Claude Code로, 데이터 레이어(SQL)는 Windows PC로 분리 개발 후 통합.

검증 성공 기준: 실제 Impala 데이터가 연결된 대시보드가 문서허브에서 접근 가능하고, 사용자 3인이 요구사항 충족 판정 시 성공
실패 시 대응: 데이터 연결 통합 오류 → Windows(SQL) ↔ AWS(웹) 데이터 전달 포맷(JSON/CSV) 표준화 후 재시도
4th

Task 4 → 자동 분석 + 인사이트 리포트 메일 발송

Phase 0 결과에 따라 범위 확정 부분 검증

AIPro/FabriX 스케줄 자동화 가능 여부에 따라 두 가지로 분기. 자동화 지원 시 전자동, 미지원 시 반자동 운영으로 진행. Task 1~3이 완성된 환경에서만 의미 있는 검증이 가능하므로 마지막 순서.

검증 성공 기준 (반자동): 담당자가 SQL 실행 후 15분 이내에 인사이트 리포트 메일을 발송할 수 있으면 성공 (기존 수동 작업 대비 80% 이상 시간 단축)
실패 시 대응: 완전 자동화 불가 → "반자동 운영 SOP" 문서화로 대체. 실제 Web/WAS 서버 투입 시 자동화 재검토
리스크 관리

실패 시나리오별 대응 체계

🚨 AIPro/FabriX가 대용량 컨텍스트를 처리 못할 경우

→ 비즈니스 로직을 단순 단위로 더 분해하여 AI에게 제공. BO 기준/솔루션 기준 등 복잡 집계 조건은 사람이 주석으로 명시하고 AI에게 채워달라는 방식(Fill-in-the-blank)으로 전환

🚨 3인 간 컨텍스트가 달라 결과물이 불일치
→ 마스터 컨텍스트 문서를 단일 URL(문서허브)로 고정. 각 Task 착수 전 3인이 동일 문서를 AI에게 입력했는지 체크리스트 확인. 불일치 발생 시 루나가 통합 조정
🚨 검증 프로젝트 자체가 예상보다 오래 걸릴 경우
→ Task 3(시뮬레이션)만이라도 완료하면 "AI 기반 시안·KPI 검증 가능"을 내부 공유 가능. 전체 4개 과제 완주보다 "단계별 가시적 성과물" 생성을 우선
