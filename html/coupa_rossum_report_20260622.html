Coupa의 Rossum 인수 사례 분석
보고 배경 및 요지

  • 인수가는 비공개이며 자금 조달은 Coupa의 현금(인수 전 FCF $174M, 이후 비공개) 또는 TB의 추가 지원을 활용했을 것으로 추정

  • [상품] 거래 문서 특화 T-LLM을 확보하여 인보이스, 구매주문서 처리 등 문서처리 자동화 역량을 강화
    • 기존 OCR(광학문자인식) 기반 방식 대비 처리 시간 최대 95% 단축, 정확도 98% 달성 (Rossum 발표 기준)
    • Rossum Aurora는 인보이스, 구매주문서 등 거래 문서에 특화된 T-LLM으로 276개 언어 및 30개 언어의 손글씨 인식을 지원
    • 단순 데이터 추출을 넘어 문서 이해, 추론, 업무 실행 기반의 AI Agent 방식 문서 자동화 제공 (상세 내용 별첨 4 참조)
  • [사업] 고객 생산성 향상 및 경쟁사 대비 기술 우위 확보
    • 인보이스 처리 자동화를 통해 업무량 90% 이상 절감하고, 향후 5년간 고객 비용 $300B 절감을 목표로 제시
    • SAP Ariba 등 주요 S2P 경쟁사 대비 IDP 기술을 선제 확보하여 차세대 조달 AI 시장에서 경쟁 우위를 확보

  • 고금리 등 환경 변화에 대응하고, 향후 기업의 Value-up을 위해 안정적인 현금창출능력 확인 → FCF, 이자상환능력 검토
  • 도메인 특화 LLM, AI Agent, 독점 데이터 등 차별화된 AI 기술 역량 보유 여부 확인
  • AI 대체 가능성이 낮은 사업 구조 평가
    • 특정 영역의 워크플로우, 데이터, 네트워크가 결합된 플랫폼
    • 독점적·폐쇄적 데이터 자산을 보유한 사업
    • 사용량(Usage) 또는 성과(Outcome) 기반 수익모델을 보유한 사업
주요 PE의 투자 포트폴리오 및 전략을 모니터링하여 투자 및 M&A 시 활용

  • SaaS 시장의 기업 매수·매도, 투자 현황 지속 파악

  • 도메인 특화 LLM, AI Agent 등 차별화된 AI 기술 역량 확보 필요

별첨 4. 기존 OCR 대비 Rossum AI Agent 차별성
Rossum의 IDP 시장 내 위상
  • Rossum은 Everest Group의 「IDP PEAK Matrix® 2026」에서 32개 공급업체 평가 중 Leader 기업으로 선정
    • 거래 문서 특화 T-LLM인 Aurora를 기반으로 문서 분할, 분류, 데이터 추출 등 핵심 IDP 기능 제공
    • 270개 이상 언어를 지원하며, 직관적 UI, AI 학습 기능, 보고 기능 등에서 높은 평가 확보
기존 OCR 대비 AI Agent 차별성
  • 기존 OCR(광학문자인식)은 문서 내 텍스트를 인식하고 정해진 규칙에 따라 데이터를 이관하는 방식으로, 문맥 이해 및 예외 상황 대응에 한계
  • 반면 AI Agent는 문서 유형과 거래 맥락을 이해하고, 기업 정책 및 업무 기준에 따라 판단·검증·후속 처리를 수행
    • 예) 인보이스 금액, 지급 조건, 조기결제 할인 여부 등을 해석하여 처리 방안 제시
    • 예) 공급업체, PO, 중복 결제 여부 등을 검증하고 이상 거래는 검토 요청
AI Agent 핵심 역량
  • [문서 이해] 복잡한 송장, 손글씨, 금융 문서, 업계별 용어 등 다양한 문서 형식과 맥락을 이해
  • [지능적 판단] 중복 결제, PO 불일치, 가격 차이, 비정상 거래 등을 자동 식별
  • [지속 학습] 공급업체별 문서 양식을 학습하고, 데이터 추출 정확도와 예외 처리 역량을 지속 개선
시사점
  • Rossum의 차별성은 단순 OCR 정확도가 아니라, 문서 이해·판단·학습을 기반으로 업무 자동화 범위를 확장하는 AI Agent 역량에 있음
  • 이에 따라 문서 자동화는 기존 '텍스트 추출' 중심에서 '문맥 이해 → 판단·검증 → 업무 실행' 중심으로 진화 중

기본 현황
공식 명칭Rossum s.r.o.
설립 / 본사'17년 / 체코 프라하 (Prague, Czech Republic)
공동 창업자Tomáš Gogár (CEO) · Petr Baudiš · Tomáš Tunys (체코기술대학교(CTU Prague) AI 연구 출신)
재무 현황매출 $44.9M ('24년 기준, YoY +37%)
직원 수약 450명 (추정, '25년 기준)
처리 실적연간 $700B+ 규모 거래 문서 처리, 누적 $1.3조 달러 처리 실적
누적 투자$104M (General Catalyst 주도, '21년 Series A)
상장 이력비상장 스타트업 → '26년 5월 Coupa Software에 피인수
최근 이력'24. 4월 Coupa와 전략적 파트너십 체결 → '26. 5월 인수로 기술 내재화 완료
사명 유래체코 극작가 Karel Čapek의 희곡 「R.U.R.(Rossum's Universal Robots, 1920년)」에서 차용 — '로봇(Robot)'이라는 단어를 세계 최초로 사용한 작품
사업 영역 및 업계 위상
  • IDP(Intelligent Document Processing, 지능형 문서처리) 전문 SaaS 스타트업
  • B2B 거래 문서(인보이스, 구매주문서, 선하증권 등 비정형 문서) AI 자동 판독·추출·검증에 특화
    • 거래 문서 특화 T-LLM(Transactional LLM) 「Rossum Aurora」를 기반으로 기존 OCR 대비 처리 시간 최대 95% 단축, 정확도 98% 달성 (Rossum 발표 기준)
  • Everest Group 「IDP PEAK Matrix® 2026」 32개 공급업체 평가 중 Leader 기업 선정
    • 직관적 UI, AI 학습 기능, 270개 이상 언어 지원 등에서 높은 평가 확보
주요 경쟁사 현황
  • ABBYY : IDP 전통 강자. 범용 OCR 역량에 AI를 결합한 FlexiCapture·Vantage 제품 보유. 비정형 거래 문서 특화 T-LLM 미보유로 Rossum 대비 정확도 열위
  • Hyperscience : 뉴욕 기반 IDP 스타트업 (누적 $190M+ 조달). 공공·금융·보험 분야 특화. 다언어·비정형 거래 문서 처리 역량 열위
  • AWS Textract : Amazon 클라우드 기반 범용 문서 추출 서비스. 비용 효율적이나 거래 맥락 이해 부족, 도메인 특화 정확도 한계
  • Google Document AI : Gemini 기반 고도화 추진 중. 범용 생성형 LLM 방식으로 금융·거래 문서 처리 시 할루시네이션(사실 왜곡 오류) 리스크 존재
  • UiPath Document Understanding : RPA(Robotic Process Automation) 워크플로우와의 통합 강점. 독립 IDP 정확도보다 기존 자동화 연동에 특화

별첨 7. Rossum 기업 개요 (계속)
제품 포트폴리오 / 핵심 기술

단일 플랫폼 내 AI 엔진·에이전트 워크플로우·학습 시스템·ERP 연동으로 구성된 End-to-End IDP 솔루션

제품 / 기능주요 기능
Rossum Aurora (T-LLM) 거래 문서 특화 Transactional LLM. 1,100만 건 이상 실거래 문서 학습 데이터 기반. Discriminative Decoder 방식으로 텍스트 생성 없이 추출만 수행 → 할루시네이션 원천 차단. 276개 언어 지원 (손글씨 30개 언어)
AI Agent (에이전트 워크플로우) 단순 데이터 추출을 넘어 문서 이해→판단→검증→업무 실행 자동화. 중복 결제·PO 불일치·가격 차이·비정상 거래 자동 탐지. 조기결제 할인 여부 판단 등 비즈니스 로직 직접 실행
Human-in-the-Loop 검증 자동화와 인간 검토를 결합한 하이브리드 처리 방식. 고객 평균 92.5% 데이터 추출 정확도 달성. 수작업 대비 90% 이상 작업량 절감. 오류율 37.6% 감소
Trainer (자기 학습) 공급업체별 문서 양식 자동 학습. 목표 정확도 도달 필요 문서 수를 기존 대비 10분의 1로 절감. 학습 누적 시 오류율 지속 감소
ERP 통합 API / SDK SAP·Oracle NetSuite·Coupa·Microsoft Dynamics 등 주요 ERP·구매 시스템 네이티브 통합. rossum-api 3.8.0·rossum-agent-client 1.1.0 SDK 제공

Aurora 1.5 (2024.10 출시) : 데이터 추출 정확도 추가 25% 향상 / 100페이지 이상 장문서 처리 속도 4배 개선

DOCile : Rossum 자체 개발 레이블 학습 데이터셋. 제3자 데이터 미활용으로 고객 데이터 외부 유출 리스크 없음

Discriminative Decoder : 언어모델이 텍스트를 새로 생성하지 않고 문서에서 정보를 직접 추출하는 방식. 생성형 방식 대비 할루시네이션 발생 구조 자체를 제거
