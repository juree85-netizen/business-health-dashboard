PROJECT PLAN
셀레요 (Selleyo)
영상 기반 요리 레시피 자동화 앱 — 프로젝트 추진 계획서
작성일: 2026-06-12 문서 버전: v1.0 상태: 초안
1. 프로젝트 개요

셀레요(Selleyo)는 유튜브 쇼츠 등 숏폼 요리 영상에서 핵심 조리 순간을 자동으로 캡처하여 단계별 레시피 카드로 변환하는 AI 기반 요리 레시피 자동화 앱입니다.

사용자가 보유한 재료를 입력하면 만들 수 있는 메뉴를 추천하고, 부족한 재료를 안내하는 재료 매칭 기능, 그리고 장기적으로 사용자가 레시피를 직접 작성하고 공유하는 커뮤니티 플랫폼으로 확장합니다.

항목 내용
앱 이름 셀레요 (Selleyo) — "설레다 + 요리"의 합성어
서비스 유형 모바일 앱 (iOS / Android), 웹 뷰어 병행
핵심 기술 영상 프레임 추출 AI, 컴퓨터 비전(조리 단계 분류), 재료 인식 NLP, 추천 알고리즘
추진 배경 숏폼 요리 영상 소비 증가에도 불구, 영상 속 레시피를 기록·재현하기 어려운 사용자 불편 존재
주요 목적 영상 → 레시피 자동 변환으로 요리 진입 장벽 낮추기 / 냉장고 재료 기반 메뉴 결정 지원
VISION

2. 핵심 문제 정의 및 타겟 사용자
# 문제 현상 및 불편사항
P1 레시피 기록의 비효율성 영상을 보며 직접 받아 적거나, 영상을 반복 재생하며 조리 단계를 파악해야 함. 숏폼 특성상 빠르게 지나가는 영상에서 정보 포착이 어려움
P2 재료와 메뉴의 단절 냉장고에 있는 재료로 어떤 요리를 할 수 있는지 파악하기 위해 별도 검색이 필요하며, 부족한 재료를 정확히 파악하기 어려움
P3 레시피 정보의 파편화 유용한 레시피가 유튜브·인스타그램·블로그 등에 분산되어 있어 나중에 재찾기 어렵고, 내가 만든 변형 레시피를 저장·공유할 공간이 없음
타겟 구분 특성 주요 니즈
1순위 (코어) 20~30대 1~2인 가구, 요리 입문자 간편하게 따라 할 수 있는 단계별 가이드 / 냉장고 재료 활용 레시피 추천
2순위 바쁜 직장인, 육아 중 부모 빠른 레시피 검색 및 저장 / 장보기 전 재료 확인
3순위 (장기) 요리 콘텐츠 크리에이터 직접 만든 레시피 공유 / 팔로워와 소통하는 커뮤니티
3. 주요 기능 상세
기능 1 · 영상 → 레시피 자동 변환
  • 유튜브/쇼츠 URL 또는 영상 공유 링크를 앱에 붙여넣기
  • AI가 조리 핵심 장면을 1~2초 단위로 자동 캡처
  • 프레임별 이미지 + 자막/음성 인식으로 단계 설명 자동 생성
  • 완성된 단계별 레시피 카드로 시각화 (슬라이드 형태)
  • 재료 목록, 소요 시간, 난이도 자동 태깅
AI 영상 분석
OCR / STT
자동 구조화
기능 2 · 냉장고 재료 기반 메뉴 추천
  • 보유 재료를 텍스트 입력 또는 사진 촬영으로 등록
  • 입력 재료와 레시피 DB를 매칭하여 만들 수 있는 메뉴 목록 추천

  • 유통기한 임박 재료를 우선 활용하는 추천 옵션

4. MVP 범위 제안 (1차 출시)

MVP는 핵심 가치인 "영상 → 레시피 자동 변환"에 집중합니다. 커뮤니티 기능과 고도화된 추천 알고리즘은 사용자 데이터 확보 이후 단계적으로 추가합니다.

6. 기대효과
U
사용자 가치
  • 영상 레시피 기록 시간 80% 단축
  • 냉장고 재료 활용률 향상 → 식재료 낭비 감소
  • 요리 입문자의 심리적 진입 장벽 해소
  • 나만의 레시피 컬렉션 관리 가능
B
비즈니스 가치
  • 숏폼 요리 콘텐츠 소비 트렌드와 완벽한 정합
  • 레시피 DB 축적 → 데이터 자산 확보
  • 크리에이터 생태계 구축으로 UGC 기반 성장
  • 식품/유통사 B2B 연동 수익 모델 확장 가능
M
시장 기회
  • 국내 숏폼 요리 영상 월간 조회 수 수십억 건
  • 밀키트·식재료 배달 시장과 자연스러운 연동
  • 기존 레시피 앱의 텍스트 중심 한계 극복
  • 일본·동남아 등 아시아 요리 문화권 확장 가능
T
기술 자산
  • 요리 영상 특화 AI 모델 독자 개발 → 진입 장벽
  • 재료–레시피 매칭 DB의 네트워크 효과
  • 사용자 행동 데이터 기반 추천 정확도 지속 향상
  • 멀티모달 AI 활용 사례로 기술 확장성 확보
7. 주요 리스크 및 대응 방안
# 리스크 영향도 발생 가능성 대응 방안
R1 유튜브 API 정책 변경 / 크롤링 제한 높음 중간 공식 YouTube Data API 활용, 다중 플랫폼(틱톡·인스타) 분산 지원, 사용자 직접 업로드 방식 병행
R2 AI 레시피 변환 정확도 미달 높음 중간 MVP 단계에서 전문 편집팀 수동 검수 병행, 사용자 피드백 기반 모델 재학습 루프 구축
R3 저작권 이슈 (영상 콘텐츠 무단 활용) 높음 중간 원본 영상 링크 기반 서비스 (영상 재호스팅 금지), 크리에이터 저작권 고지 명시, 법률 자문 확보
R4 초기 사용자 획득 비용 과다 중간 중간 요리 유튜버·인플루언서와 초기 파트너십 체결, 공유 기능 기반 바이럴 성장 전략 우선
R5 경쟁 앱(만개의 레시피 등)의 유사 기능 출시 중간 낮음 영상 자동 변환 AI 기술의 빠른 개발로 선점 효과 확보, UGC 커뮤니티 기반 Lock-in 강화
R6 개발 일정 지연 (AI 모델 개발 난이도) 중간 중간 MVP는 룰 기반 프레임 추출로 먼저 출시 후 AI 모델 점진적 교체, 오픈소스 모델 활용으로 개발 기간 단축
