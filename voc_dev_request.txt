■ 1. 요청 배경

현재 solution_if_voc 테이블을 통해 CS 대시보드로부터 VoC 건수·불만건수·실사용자수
3개 지표를 월별 집계(CNT) 형태로 수신하고 있습니다.

사업건전성 대시보드 고도화를 위해 아래 3개 신규 지표 산출이 필요하나,
기존 월별 집계 데이터로는 산출이 불가합니다.

  #  신규 지표                        기존 데이터로 산출 불가 이유
  ─────────────────────────────────────────────────────────
  1  접수월별 개선 요청 건수            요구유형 필드 없음
  2  접수월별 평균 처리 리드타임        건별 요청·완료 일시 없음
  3  월별 지연 건수                    건별 예정·완료 일시 없음

이에 CS 대시보드로부터 건별 원천 데이터를 수신·저장하는 신규 테이블과,
시점에 따라 값이 변하는 지표를 고정 관리하기 위한 스냅샷 테이블 개발을 요청합니다.

※ CS 대시보드 측 신규 API 개발은 CS팀에 별도 요청 예정입니다.
   본 요청은 해당 API로부터 데이터를 수신하여 저장·가공하는 당사 내부 개발 범위입니다.


■ 2. 요청 사항 요약

  #  구분              내용
  ─────────────────────────────────────────────────────────────────────
  1  기존 테이블       solution_if_voc — 현행 유지 (변경 없음)
  2  신규 개발 ①      solution_if_voc_detail — CS 건별 데이터 수신 및 저장 배치 개발
  3  신규 개발 ②      sol_if_voc_snap — 월별 지표 스냅샷 테이블 생성 및 집계 배치 개발


■ 3. 신규 개발 ①  solution_if_voc_detail

  CS 대시보드 건별 원천 데이터를 수신하여 저장하는 테이블.
  솔루션사업부가 직접 집계·분석하여 신규 지표를 산출하는 기반 데이터로 활용합니다.

■ 4. 신규 개발 ②  sol_if_voc_snap (Datalake 스냅샷 테이블)

  [4-1. 테이블 역할]
  리드타임·지연건수는 계산 시점에 따라 값이 달라지는 지표이므로,
  매월 고정 시점에 계산한 값을 저장하여 일관된 추이를 관리합니다.

  ※ 개선 요청 건수는 접수 시점 값이 고정되므로 스냅샷 불필요
    → solution_if_voc_detail에서 상시 재계산합니다.

  [4-2. 테이블 필드]

   #  필드명              설명
  ────────────────────────────────────────────────────────────────────
   1  기준월              지표 산출 기준 월 (예: 2026-03)
   2  스냅샷 일자         실제 계산·저장 일자 (예: 2026-04-01)
   3  상품/서비스         대상 상품명
   4  평균 리드타임(시간) 해당 기준월 완료건 기준 평균 처리 시간
   5  지연건수            해당 기준월 기준 지연 건수

  [4-3. 집계 산출식]

  ▷ 평균 리드타임

    평균 리드타임(M월) =
      SUM(완료일자 - 요청일자)
      ÷ COUNT(VOC_ID)
      WHERE 기준월 = M
        AND 진행단계 = '완료'
        AND 상품/서비스 = [상품명]

  ▷ 지연건수

    지연건수(M월) = COUNT(VOC_ID)
    WHERE 상품/서비스 = [상품명]
      AND (
        완료일자 > 예정일자                           -- 완료됐으나 기한 초과한 건
        OR (완료일자 IS NULL AND 예정일자 < M월 말일) -- 미완료이며 기한이 지난 건
      )

  [4-4. 배치 적재 조건]

   #  항목        내용
  ──────────────────────────────────────────────────────────────────────
   1  산출 시점   매월 1일
   2  산출 기준   M-1월 기준 계산
   3  적재 방식   M-1월 계산값을 1행씩 추가 적재 (기존 누적 데이터 유지)
