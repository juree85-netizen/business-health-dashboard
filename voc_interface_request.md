# VoC 데이터 인터페이스 신규 개발 요청서

---

## 0. 배경

- 기존 `solution_if_voc` 테이블은 DATALAKE API(`/datalake/QualitySummary`)를 통해 VOC건수·불만건수·실사용자수 3개 지표를 월별 집계(CNT) 형태로 수신 중 (배치 주기: 1일 1회)
- 기존 데이터는 월별 합산값(PROD3_NM / GUBUN / YYYY / MM / CNT 구조)이므로, 개별 건 단위 분석(리드타임·지연 여부 등) 불가
- 신규 지표(접수월별 개선건수·평균 리드타임·지연건수) 산출을 위해 CS 대시보드 원천의 RAW 데이터 추가 필요
- **인터페이스 방식 확정: 신규 테이블(RAW 데이터) 방식 채택**
  - 기존 테이블(`solution_if_voc`): 현행 유지
  - 신규 테이블(`solution_if_voc_detail`): VoC 건별 RAW 데이터 적재 → 당사가 직접 집계 및 분석
  - 채택 이유: 상품별·채널별·처리구분별 세부 분석 및 향후 분석 확장성 확보
  - 기존 창구(`/datalake/QualitySummary`)는 그대로 유지, 신규 데이터 제공 창구(API) 추가 개발 요청
  - 기존 파라미터의 조회 범위 1년 이내 제한 해제 필요 (당사 요청 범위: 2023년 1월~현재, 약 3년치)
  - 신규 창구가 필요한 이유: 기존 창구는 월별 집계 구조로, RAW 데이터(16개 필드)와 구조가 근본적으로 달라 기존 테이블에 단순 추가 불가
- CS 대시보드는 현재 시점 스냅샷 방식 운영 중. 당사는 시계열 트렌드 분석 목적이므로 매월 1회 Full 재집계 방식으로 제공 요청

## 1. 요청 개요

| 항목 | 내용 |
|------|------|
| 요청 목적 | Solution Biz Health Check 대시보드 VoC 지표 산출을 위한 건별 원천 데이터 신규 제공 |
| 활용 시스템 | 사업건전성 대시보드 (`solution_if_voc_detail` 테이블, `solution_if_voc_snapshot` 테이블) |
| 원천 시스템 | CS 대시보드 |
| 기존 인터페이스 | `/datalake/QualitySummary` (사용자수 / VOC건수 / 불만건수) — **현행 유지** |

---

## 2. 요청 사항 요약

| 구분 | 내용 |
|------|------|
| 기존 인터페이스 | `/datalake/QualitySummary` → `solution_if_voc` 현행 유지 (변경 없음) |
| 신규 인터페이스 ① | 건별 원천 데이터 16개 필드 → `solution_if_voc_detail` **신규 개발 요청** (아래 3항 참조) |
| 신규 인터페이스 ② | 누계지연건수 → `solution_if_voc_snapshot` **신규 개발 요청** (아래 5항 참조) |
| 지표 산출 | 수신된 건별 데이터를 대시보드에서 직접 집계하여 3개 지표 산출 |

## 5. 누계지연건수 스냅샷 테이블 (`solution_if_voc_snapshot`)

월별 지연 추이 관리를 위해 **CS 대시보드에서 당월 기준 누계지연건수를 매일 집계하여 제공**합니다.
당사는 수신된 값의 당월 행을 삭제 후 재적재하며, 월이 바뀌면 이전 월 데이터는 그대로 유지됩니다.

**지연 정의**
> 진행단계 불문, 제공 시점 기준으로 예정일자를 초과한 모든 미처리 잔여 건

**스냅샷 테이블 필드**

| # | 필드명 | 설명 |
|---|--------|------|
| 1 | 기준월 | 데이터 제공 기준 월 (예: 2026-04) |
| 2 | 기준일자 | 데이터 제공 기준 일자 (예: 2026-04-15) |
| 3 | 상품/서비스 | 대상 상품명 |
| 4 | 누계지연건수 | 기준일자 현재 당월 기준 전체 잔여 지연 건수 |

**데이터 제공 조건**

| 항목 | 내용 |
|------|------|
| 집계 기준 | 제공 시점 현재, 당월 기준, 상품/서비스별 |
| 제공 주기 | 매일 |
| 적재 방식 | 당월 행 삭제 후 재적재 (과거 월 데이터 유지) |
| 보관 범위 | 2023년 1월 ~ 계속 누적 |

## 6. 산출 예정 지표

수신된 건별 데이터 및 스냅샷 테이블을 활용하여 아래 3개 지표 산출

| # | 지표명 | 계산식 | 활용 테이블 / 필드 |
|---|--------|--------|-------------------|
| 1 | 접수월별 개선 요청 건수 | 요구유형 = '개선'인 건수 (기준월별, 상품별) | `solution_if_voc_detail` — 기준월, 상품/서비스, 요구유형 |
| 2 | 접수월별 평균 처리 리드타임 | SUM(리드타임(시간)) ÷ COUNT(진행단계='완료') | `solution_if_voc_detail` — 기준월, 상품/서비스, 진행단계, 리드타임(시간) |
| 3 | 월별 지연 건수 | 스냅샷기준월별 누계지연건수 | `solution_if_voc_snapshot` — 기준월, 상품/서비스, 누계지연건수 |

## 7. 대시보드 분석 필터 활용 필드

지표 산출 외에, 아래 2개 필드는 대시보드 내 **분석 필터(drill-down 차원)** 로 활용됩니다.
별도 테이블 추가 없이 `solution_if_voc_detail` 건별 데이터에서 직접 집계합니다.

| # | 필드명 | 활용 방식 | 값 예시 |
|---|--------|----------|---------|
| 1 | 접수채널 | 채널별 VoC 건수 / 지표 필터링 | CALL / CHAT / E-MAIL / WEB |
| 2 | 처리구분 | 처리 선별 VoC 건수 / 지표 필터링 | 1선 / 2선 / 3선 |
